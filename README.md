# ZimRecruit 🇿🇼

> **Blockchain-powered recruitment & credential verification platform for Zimbabwe.**  
> Every credential on ZimRecruit carries a cryptographic, on-chain proof of authenticity recorded by accredited institutions on the local credential chain.

---

## Overview

| Layer            | Technology                          |
|------------------|-------------------------------------|
| Frontend         | Next.js 14 (App Router), Tailwind CSS, Zustand |
| Backend API      | Node.js + Express + TypeScript      |
| Auth / Media Storage | Appwrite Auth + one Storage bucket |
| Database         | MySQL 8.0 + Redis 7                 |
| Blockchain       | Local Hardhat mockchain only · Solidity 0.8.24 |
| Smart Contract   | `ZimRecruitRegistry.sol` (AccessControl) |
| Email            | Nodemailer (Gmail App Password)     |
| Deployment       | Docker Compose · Nginx · GitHub Actions CI/CD |

---

## Architecture

```
Browser (Next.js)
      │ HTTPS
      ▼
   Nginx (reverse proxy + rate limit)
      ├─▶ /api/*  → Express API (Node.js)
      │              ├─ MySQL  (user data, jobs, docs)
      │              ├─ Redis  (BullMQ, JWT deny-list)
      │              ├─ Appwrite (auth + single media bucket)
      │              └─ Local RPC  (read/write contract)
      └─▶ /*      → Next.js (SSR/SSG)
```

## Quick Start (Local Dev)

For a full production guide covering deployment, MySQL, blockchain, and Appwrite, see [`docs/DEPLOYMENT_GUIDE.md`](docs/DEPLOYMENT_GUIDE.md).

### Prerequisites
- Node.js 22+
- Docker & Docker Compose
- An [Appwrite](https://cloud.appwrite.io) project (free tier works)
- A local Hardhat mockchain wallet for development

### 1. Clone & install

```bash
git clone https://github.com/yourname/zimrecruit.git
cd zimrecruit
npm install
```

### 2. Environment

```bash
cp .env.example .env
# Fill in all values — see comments in .env.example
```

**Minimum required variables for local dev:**

| Variable              | Where to get it                                    |
|-----------------------|----------------------------------------------------|
| `MYSQL_PASSWORD`      | Choose any strong password                         |
| `APPWRITE_PROJECT_ID` | Appwrite console → Project settings                |
| `APPWRITE_API_KEY`    | Appwrite console → API Keys → Server key           |
| `APPWRITE_BUCKET_MEDIA` | One Appwrite Storage bucket ID, e.g. `zimrecruit-media` |
| `JWT_PRIVATE_KEY_PEM` | `openssl genrsa -out private.pem 4096`             |
| `JWT_PUBLIC_KEY_PEM`  | `openssl rsa -in private.pem -pubout -out pub.pem` |
| `SMTP_USER/PASS`      | Gmail → Security → App Passwords (2FA required)    |
| `VERIFIER_PRIVATE_KEY`| Hardhat account #1 from `mockchain/start.sh`        |
| `CONTRACT_ADDRESS`    | `mockchain/deployments/localhost.env` after step 4 |

### 3. Start infrastructure

```bash
cd infra/docker
docker compose up mysql redis -d
# Wait for healthy status
docker compose ps
```

### 4. Start the local credential mockchain

```bash
cd mockchain
npm install
bash ./start.sh
# Copy values from mockchain/deployments/localhost.env into your .env
```

The app supports only this local Hardhat mockchain. Blockchain env values should always use:

```env
BLOCKCHAIN_CHAIN_ID=31337
BLOCKCHAIN_RPC_URL=https://skirmish-thicken-derived.ngrok-free.dev
```

Use the deployed `CONTRACT_ADDRESS`, `VERIFIER_PRIVATE_KEY`, and `VERIFIER_WALLET_ADDRESS` generated by the mockchain script. Do not configure any public RPC network for this project.

### 4b. Expose the mockchain with ngrok

Use ngrok only when a live web/API environment needs to call your local Hardhat RPC.

1. Install ngrok from `https://ngrok.com/download`.
2. Authenticate once:

```bash
ngrok config add-authtoken YOUR_NGROK_AUTH_TOKEN
```

3. Keep the mockchain running in one terminal:

```bash
cd mockchain
bash ./start.sh
```

4. Open a second terminal and expose port `8545`:

```bash
ngrok http 8545
```

5. Use the fixed HTTPS forwarding URL for this app:

```env
BLOCKCHAIN_RPC_URL=https://skirmish-thicken-derived.ngrok-free.dev
```

Keep `BLOCKCHAIN_CHAIN_ID=31337` and keep the same mockchain contract and verifier values.

For the longer checklist, see [`docs/MOCKCHAIN_NGROK_GUIDE.md`](docs/MOCKCHAIN_NGROK_GUIDE.md).

### 5. Run database migrations

```bash
mysql -h 127.0.0.1 -u zimrecruit -p zimrecruit < infra/docker/schema.sql
npm run db:check
```

### 5b. Seed an admin user

Create the admin account in Appwrite Auth first, then set `ADMIN_APPWRITE_USER_ID`, `ADMIN_EMAIL`, and `ADMIN_FULL_NAME` in `.env`. For first bootstrap only, you can set `ADMIN_PASSWORD` and the seed script will create the Appwrite user.

```bash
npm run seed:admin
npm run db:check
```

### 6. Start dev servers

```bash
# From repo root — starts both API and Next.js with hot reload
npm run dev
```

- **API:**     http://localhost:4000
- **Web:**     http://localhost:3000
- **Health:**  http://localhost:4000/health

## Appwrite Setup

### Buckets (create in Appwrite console)

| Bucket ID           | Purpose                             | Max size | Allowed types         |
|---------------------|-------------------------------------|----------|-----------------------|
| `zimrecruit-media`  | Single bucket for avatars, logos, images, and uploaded credential files | 10 MB | PDF, PNG, JPEG, WEBP, SVG |

### Bucket permissions (`zimrecruit-media`)
```
Create: authenticated users
Read:   none at bucket level; backend streams files after role checks
Delete: backend/admin only
Update: none
```

### Appwrite Auth
- Enable **Email/Password** provider
- Set redirect URLs: `http://localhost:3000`, `https://zimrecruit.co.zw`
- Do not create Appwrite Databases for platform records; MySQL is the data source of truth.

## API Reference

Base URL: `https://zimrecruit.co.zw/api` (or `http://localhost:4000/api`)

All protected routes require:
```
Authorization: Bearer <accessToken>
```

### Auth
| Method | Path             | Auth | Description                        |
|--------|------------------|------|------------------------------------|
| POST   | /auth/register   | ❌   | Register (Appwrite UID required)   |
| POST   | /auth/login      | ❌   | Exchange Appwrite session for JWT  |
| POST   | /auth/logout     | ✅   | Invalidate session                 |

### Jobs
| Method | Path                   | Auth              | Description           |
|--------|------------------------|-------------------|-----------------------|
| GET    | /jobs                  | ❌                | List / search jobs    |
| GET    | /jobs/:id              | ❌                | Job details           |
| POST   | /jobs                  | employer, admin   | Create job            |
| PATCH  | /jobs/:id              | employer, admin   | Update job            |
| GET    | /jobs/:id/applicants   | employer, admin   | Verified pipeline     |

### Documents
| Method | Path                      | Auth            | Description            |
|--------|---------------------------|-----------------|------------------------|
| GET    | /documents                | applicant       | List own documents     |
| POST   | /documents                | applicant       | Register + hash a file |
| POST   | /documents/:id/verify     | applicant       | Send to institution    |
| GET    | /documents/:id/stream     | applicant/verifier | Serve file            |

### Verifications
| Method | Path                       | Auth     | Description              |
|--------|----------------------------|----------|--------------------------|
| GET    | /verifications/queue       | verifier | Pending docs for my inst |
| POST   | /verifications/:id/approve | verifier | Record approved credential |
| POST   | /verifications/:id/reject  | verifier | Reject with reason       |

### Public
| Method | Path                    | Auth | Description               |
|--------|-------------------------|------|---------------------------|
| GET    | /public/verify/:hash    | ❌   | Look up on-chain attestation |

## Smart Contract

### `ZimRecruitRegistry` — Local Hardhat Mockchain

```solidity
function verify(bytes32 documentHash, uint256 institutionId, string institutionName) external;
function revoke(bytes32 documentHash) external;
function get(bytes32 documentHash) external view returns (Attestation memory);
function isValid(bytes32 documentHash) external view returns (bool);
```

- Only addresses with `VERIFIER_ROLE` can attest or revoke.
- `DEFAULT_ADMIN_ROLE` is held by `PLATFORM_ADMIN_ADDRESS` in the mockchain deployment.
- No personal data is ever stored on-chain — only the document SHA-256 hash.
- Duplicate attestion is blocked at the contract level.
- Supported RPC target: `https://skirmish-thicken-derived.ngrok-free.dev`, forwarding to the local mockchain on port `8545`.

### Run contract tests

```bash
cd contracts
npx hardhat test
# With gas report:
REPORT_GAS=true npx hardhat test
```

## Deployment (Production)

### 1. Server requirements
- Ubuntu 24.04 VPS — minimum 2 vCPU, 4 GB RAM
- Docker + Docker Compose installed
- Domain pointed to VPS IP

### 2. Deploy

```bash
scp -r infra/docker deploy@your-vps:/opt/zimrecruit
ssh deploy@your-vps
cd /opt/zimrecruit
cp .env.example .env  # fill in production values
docker compose up -d
```

### 3. SSL (Let's Encrypt)

```bash
apt install certbot python3-certbot-nginx
certbot --nginx -d zimrecruit.co.zw -d www.zimrecruit.co.zw
```

Update `nginx.conf` to uncomment the HTTPS redirect.

### 4. CI/CD

Push to `main` triggers:
1. Lint + typecheck
2. API unit tests (against MySQL test service)
3. Smart contract tests (Hardhat)
4. Docker build & push to GHCR
5. SSH deploy to VPS (`docker compose pull && up -d`)

Set these GitHub Secrets: `VPS_HOST`, `VPS_USER`, `VPS_SSH_KEY`, `NEXT_PUBLIC_API_URL`, `NEXT_PUBLIC_APPWRITE_ENDPOINT`, `NEXT_PUBLIC_APPWRITE_PROJECT`, `NEXT_PUBLIC_APPWRITE_BUCKET_MEDIA`.

## Security Notes

- **Private keys** (`VERIFIER_PRIVATE_KEY`) must never be committed. Add `.env` to `.gitignore`.
- **JWT RS256** keypair: use 4096-bit RSA. Rotate every 90 days.
- **Verifier wallet** should use a local development key while testing.
- **Appwrite API Key** is server-only. Never expose in the browser bundle.
- Documents are always hashed **server-side** — the client cannot supply a forged hash.
- Audit log rows are hash-chained (`row_hash = sha256(prev_hash + row_data)`) for tamper evidence.
- Rate limiting: 5 auth attempts / 15 min, 100 API requests / min.
- CORS: restricted to `APP_BASE_URL`.
- CSP, HSTS, X-Frame-Options, nosniff all enforced via Helmet + Nginx.
